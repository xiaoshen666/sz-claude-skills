# Process Design Guide

## Prerequisites:
- Requirement analysis document
- Functional architecture design results (core functional modules and submodule lists)

## Output Types:
- **Process Architecture Diagram**: Show overall process framework and key process nodes
- **Key Process Swimlane Diagram**: Show detailed steps and responsibility division of specific processes
- **System Flow Diagram**: Design system processes based on functional modules

## Output Tools:
All outputs must be displayed using Obsidian.

## Design Method:
### Process Architecture Design:
1. Directly output process architecture based on functional modules
2. Define key process nodes and flow relationships in the process architecture
3. Identify and define key business processes
4. Output swimlane diagrams for key processes

### System Flow Diagram Design:
- Connect functional modules and sub-functions into flow diagrams
- Mark flow logic and core parameters on the connections between modules

## Output Format:
- Process architecture diagram: Show overall process framework and key process nodes
- Key process swimlane diagram: Show detailed steps and responsibility division of specific processes
- Visualize using Obsidian Canvas
