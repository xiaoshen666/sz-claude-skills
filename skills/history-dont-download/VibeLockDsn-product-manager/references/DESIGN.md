# Product Design Guide

## Prerequisites:
All output from previous processes.

## Output Tools:
- **Feature List**: Display using Obsidian
- **Design Specification Document**: Display using Obsidian
- **Product Prototype**: Draw using Pencil software (via MCP channel)

## Must Follow:
- The feature list must be planned based on module and submodule lists, not from other sources.
- **Must first list the feature list and obtain confirmation before entering prototype design.**
- Product prototypes must cover all pages and interaction scenarios, not based on partial modules.

## Design Steps:
### Feature List:
- Plan pages based on module and submodule lists
- Define core operations and interaction logic for each page
- Submit for user confirmation

### Design Specification Development:
- Refer to impeccable frontend-design skill to obtain design specifications
- Collect design context (target users, usage scenarios, brand personality)
- Determine design style direction (minimalist, luxurious, industrial, retro-futuristic, etc.)
- Develop typography, color, layout, animation specifications
- Output design specification document for Pencil design reference

### Product Prototype:
- Draw prototypes in Pencil based on design specifications
- Ensure design style, typography, color, etc. meet specification requirements
- Cover all pages and interaction scenarios
- Ensure prototypes match the feature list completely

## Output Format:
### Feature List Table:
| Module/Submodule | Page Name | Page Description | Core Operations | User Roles |
|------------------|-----------|-----------------|----------------|------------|

### Design Specification Document:
- **Design Style Direction**: Clear design style description
- **Typography Specifications**: Font selection, font size hierarchy, line height, font weight
- **Color Specifications**: Primary colors, secondary colors, neutral colors, text colors
- **Layout Specifications**: Grid system, spacing standards, alignment methods
- **Component Specifications**: Design specifications for buttons, forms, cards, etc.
- **Animation Specifications**: Transition effects, animation duration, interaction feedback

### Product Prototype:
- Drawn using Pencil software
- Visual effects implemented based on design specifications
- Delivered through specified MCP channel
