# Requirement Analysis Guide

## Output Tools:
All outputs must be displayed using Obsidian.

## Execution Steps:
- Analyze user requirements, clarify design scope (complete system or functional modules)
- If it's a complete system, confirm whether organization user permission management and login/registration modules are needed
- Confirm core roles and their requirements
- Confirm terminals involved in the system (mobile/web)
- Reference similar system features to supplement requirements

## Output Format:
### Requirement Document:
- Requirement description: Summarize requirement points
- Platforms and roles: Table output (application platforms, user roles, platform descriptions)
- Requirement list: Table output (requirements, requirement descriptions)

## Obsidian Tool Reference:
| Tool Name | Core Function |
|-----------|---------------|
| defuddle | Clean web page/video content, extract main text and convert to Markdown |
| json-canvas | Generate/edit Canvas canvas, create mind maps, knowledge graphs |
| obsidian-bases | Operate Bases database views, convert notes into filterable tables/cards |
| obsidian-cli | Command line interface for automated knowledge base creation, search, organization |
| obsidian-markdown | Generate Markdown notes that comply with Obsidian syntax, supporting internal links, callouts, etc. |
