# Interaction Flow Design Guide

## Prerequisites:
- Functional architecture design results
- Product prototype design results

## Output Tools:
All outputs must be displayed using Obsidian.

## Design Principles:
- Design interaction flows based on functional modules and page operation lists
- Clearly identify interaction participants and message passing order
- Define timing relationships and dependency conditions for interaction steps
- Ensure completeness and accuracy of interaction flows

## Design Steps:
1. Analyze business scenarios of key functional modules
2. Identify interaction participants (users, systems, external services, etc.)
3. Organize interaction sequences and message passing flows
4. Define timing relationships and dependency relationships in interactions
5. Draw interaction sequence diagrams using Canvas
6. Output interaction flow documents

## Output Format:
### Interaction Flow Document:
- Interaction sequence diagrams (visualized using Canvas)
- Interaction flow table:
| Interaction Step | Participant | Operation/Message | Timing Relationship | Dependency Condition |
|-----------------|-------------|-------------------|---------------------|---------------------|
