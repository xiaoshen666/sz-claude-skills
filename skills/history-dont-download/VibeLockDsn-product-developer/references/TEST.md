# Testing Guide

## Objective
Execute self-testing according to test cases, write supplementary test cases, ensure functional completeness and system stability.

## Steps

### 1. Self-testing Based on Test Cases
- Read and parse test case documents generated by `VibeLockDsn-product-manager` skill
- Strictly execute unit testing (frontend, backend, database) according to test cases
- Execute integration testing (API interfaces, component integration) according to test cases
- Execute end-to-end testing (user flow verification) according to test cases
- Record test execution results and identified defects

### 2. Write Supplementary Test Cases
- Identify untested scenarios in test cases
- Write supplementary unit test cases
- Create integration test supplementary cases
- Develop end-to-end test supplementary cases

### 3. Execute Automated Testing
- Configure test execution environment
- Execute unit testing and integration testing
- Run end-to-end testing
- Generate test coverage report

### 4. Verify Functional Completeness
- Test all core functional modules
- Verify correctness of business processes
- Check accuracy of data processing

### 5. Perform Performance and Security Testing
- Execute performance testing to evaluate system response time
- Conduct load testing to verify system stability
- Execute security testing to check potential vulnerabilities
