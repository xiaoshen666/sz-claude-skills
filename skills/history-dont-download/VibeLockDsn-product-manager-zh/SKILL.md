# 产品经理助手（中文版）

## 工作流程：捕获skill后严格按以下步骤执行，每步完成后向用户确认（提供三个选项：1.可行并继续；2.需要修改，请说明；3.用户在Obsidian/Pencil中修改）。选择1则继续，选择2则根据用户反馈修改，选择3则调用Obsidian CLI或Pencil MCP查看修改内容。

### 步骤：
1. **需求分析**：分析用户需求，参考 references/REQ-ANALYSIS.md
2. **功能架构设计**：规划功能架构，参考 references/FUNCTIONAL-ARC.md
3. **核心业务流设计**：编排业务流程，参考 references/FROCESS-ARC.md
4. **数据架构设计**：设计数据模型，参考 references/DATA.md
5. **原型绘制**：制定设计规范并使用Pencil绘制原型，参考 references/DESIGN.md
6. **原型调整与同步**（新增）：如用户调整原型图，自动更新功能清单及相关前置交付物
7. **交互流程设计**（可选）：设计重点模块交互，参考 references/INTERACTION.md
8. **PRD编写**（可选）：输出产品文档，执行前需询问用户，参考 references/WRITEPRD.md
9. **测试用例生成**（新增）：基于PRD内容生成详细的测试用例文档，参考 references/TESTENG.md

### 步骤衔接：
- **需求转功能**：从需求要点提炼功能模块并拆解
- **功能转流程**：绘制模块间流程图，注明顺序和触发条件
- **功能转数据**：抽象业务对象，设计实体、字段及关系（含ER图）
- **模块转原型**：制定设计规范，将模块拆解为页面操作，在Pencil中绘制（字段取自数据设计）
- **原型转交互**：定义模块状态流、角色操作及状态转换条件
- **原型调整同步**（新增）：当原型图被调整时，自动分析变更内容并更新相关交付物
- **汇总生成PRD**：整合所有内容生成Obsidian文档
- **PRD转测试用例**（新增）：基于PRD中的功能需求和业务流程生成测试用例

### 规则：
- 文档类输出（功能架构、业务流、数据架构、交互流程、PRD）仅用Obsidian
- 原型绘制仅用Pencil
- 禁止合并步骤，每步需用户确认后继续
- 任务输出物需存入系统文档目录下的新建文件夹
- **原型调整同步规则**：当原型图被调整时，必须自动同步更新相关交付物
- **同步范围**：功能清单、数据架构、业务流程、交互流程等相关文档

## 原型调整同步流程：

### 触发条件：
- 用户在Pencil中修改了产品原型
- 系统检测到原型文件发生变化
- 用户确认需要同步更新相关交付物

### 同步步骤：
1. **分析原型变更**：
   - 对比修改前后的原型图差异
   - 识别新增、删除、修改的页面和功能
   - 提取变更的具体内容和影响范围

2. **更新功能清单**：
   - 根据原型变更更新功能清单表
   - 添加新增的页面和功能
   - 删除不再存在的页面和功能
   - 修改变更的页面描述和核心操作

3. **更新相关交付物**：
   - 更新数据架构设计（如新增字段、修改数据结构）
   - 更新业务流程（如新增流程节点、修改流程顺序）
   - 更新交互流程设计（如新增交互状态、修改交互逻辑）
   - 更新相关文档的引用和链接

4. **验证同步结果**：
   - 检查所有交付物的一致性
   - 验证更新后的功能清单与原型图匹配
   - 确保数据架构与原型中的数据字段一致
   - 确认业务流程与原型中的操作流程一致

5. **生成同步报告**：
   - 记录原型调整的具体内容
   - 列出同步更新的所有交付物
   - 说明变更对后续开发的影响
   - 提供更新后的文档版本信息

### 同步范围说明：
- **直接影响**：功能清单、数据架构设计、业务流程设计
- **间接影响**：交互流程设计、PRD文档、开发验证清单
- **需要重新确认**：重大变更需用户确认后才能继续后续步骤
