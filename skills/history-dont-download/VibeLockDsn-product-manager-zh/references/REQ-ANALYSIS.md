# 需求分析指南

## 输出工具：
所有输出必须使用Obsidian展示。

## 执行步骤：
- 分析用户需求，明确设计范围（完整系统或功能模块）
- 若为完整系统，确认是否需要组织机构用户权限管理及登录注册模块
- 确认核心角色及其需求
- 确认系统涉及的终端（移动端/WEB端）
- 参考同类系统特性补充需求

## 输出格式：
### 需求文档：
- 需求描述：总结需求要点
- 端及角色：表格输出（应用端、使用角色、应用端描述）
- 需求清单：表格输出（需求、需求说明）

## Obsidian工具参考：
| 工具名称            | 核心作用                                                         |
|-------------------|----------------------------------------------------------------|
| defuddle          | 清洗网页/视频内容，提取正文转为Markdown                             |
| json-canvas       | 生成/编辑Canvas画布，制作思维导图、知识图谱                           |
| obsidian-bases    | 操作Bases数据库视图，将笔记转为可筛选表格/卡片                         |
| obsidian-cli      | 命令行接口，实现知识库自动化创建、搜索、整理                           |
| obsidian-markdown | 生成符合Obsidian语法的Markdown笔记，支持内部链接、提示块等原生特性        |
