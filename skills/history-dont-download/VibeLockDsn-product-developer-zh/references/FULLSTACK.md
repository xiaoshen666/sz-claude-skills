# 全栈开发指南

## 目标
使用指定技术栈（前端：React，后端：Python + FastAPI，数据库：MySQL）完成全栈开发流程。

## 一、数据库设计与实现（MySQL）

### 1. 创建数据库表结构
- 根据PRD中的数据架构设计MySQL表结构
- 定义表字段、数据类型（MySQL支持的类型）和约束
- 设置表之间的关系和外键约束

### 2. 编写数据库迁移脚本
- 创建初始表结构迁移脚本
- 编写索引和约束创建脚本
- 配置迁移工具（推荐使用Alembic，FastAPI项目常用）

### 3. 设置数据库索引和约束
- 为经常查询的字段创建索引
- 设置唯一性约束和外键约束
- 配置数据验证规则

### 4. 创建种子数据和测试数据
- 准备基础数据（如系统配置、初始用户等）
- 创建测试数据生成脚本
- 确保数据覆盖各种业务场景

### 5. 生成演示数据集
- 创建真实模拟的用户数据
- 生成业务数据样本
- 确保数据覆盖边界情况和异常场景

### 6. 创建数据导入工具
- 编写数据导入脚本
- 配置批量导入功能
- 设置数据验证和错误处理

## 二、后端开发（Python + FastAPI）

### 1. 创建API接口
- 使用FastAPI框架设计RESTful API接口规范
- 实现路由和端点处理函数
- 使用Pydantic进行请求参数验证

### 2. 实现业务逻辑层
- 创建服务层处理业务逻辑
- 使用SQLAlchemy实现数据访问层
- 配置事务管理和数据库会话

### 3. 设置认证和授权机制
- 使用JWT（JSON Web Tokens）实现用户认证
- 设置基于角色的访问控制（RBAC）
- 配置FastAPI依赖注入进行权限验证

### 4. 实现数据验证和错误处理
- 使用Pydantic模型进行请求数据验证
- 实现统一的异常处理机制
- 设置标准化的错误响应格式

### 5. 配置日志和监控
- 使用Python日志模块设置日志记录系统
- 配置性能监控和API请求跟踪
- 实现健康检查端点和系统状态监控

## 三、前端开发（React）

### 1. 创建基础组件
- 使用React函数组件创建UI组件
- 应用设计系统变量和样式规范
- 使用React Hooks管理组件状态

### 2. 实现组件交互逻辑
- 添加事件处理和用户交互
- 使用React Hook Form实现表单验证和提交
- 设置组件动画和过渡效果（使用CSS或动画库）

### 3. 开发页面组件
- 组装基础组件创建页面
- 使用React Router实现页面导航和路由
- 配置页面布局和响应式设计

### 4. 集成API调用
- 创建API服务层（使用axios或fetch）
- 实现数据获取和提交
- 设置加载状态和错误处理

### 5. 实现状态管理
- 配置状态管理库（推荐使用Redux Toolkit或React Context）
- 创建状态存储和操作
- 实现状态持久化（可选，使用localStorage或sessionStorage）

## 全栈开发流程集成

### 开发顺序建议
1. **数据库设计与实现**：先完成数据模型设计，为后续开发奠定基础
2. **后端开发**：基于数据库模型实现API接口和业务逻辑
3. **前端开发**：集成后端API，实现用户界面和交互

### 开发注意事项
- 保持前后端API接口的一致性
- 确保数据模型在前后端的统一理解
- 实现完整的错误处理和异常情况处理
- 保持代码质量和可维护性
- 进行充分的测试验证
