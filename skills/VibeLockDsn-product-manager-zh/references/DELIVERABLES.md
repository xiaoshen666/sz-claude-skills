## 三、产物检查规范

### 产物完整性检查
每个步骤完成后，检查该步骤的产物是否完整：
```
1. 列出该步骤应有的所有产物
2. 逐一检查产物是否存在
3. 检查产物内容是否完整
4. 记录缺失或不完整的产物
5. 补充缺失产物后再继续
```

### 产物质量检查
- **需求分析**：需求描述是否清晰、端及角色是否完整
- **功能架构**：模块划分是否合理、优先级是否明确
- **模块数据架构**：实体是否完整、字段是否准确
- **模块业务流程**：流程是否完整、角色职责是否清晰
- **原型设计**：是否符合功能清单、是否符合数据架构
- **PRD**：是否包含所有必要章节、是否可供开发使用

### 产物依赖检查
```
功能清单 → 数据架构设计：功能清单必须先确认
数据架构设计 → 业务流程设计：数据实体必须先确认
业务流程设计 → 原型设计：业务流程必须先确认
原型设计 → PRD：原型必须先完成
PRD → 测试用例：PRD必须先完成
```
