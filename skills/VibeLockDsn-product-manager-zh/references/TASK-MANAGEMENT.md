# 任务进度管理规范

## 一、进度文件概述

| 文件 | 位置 | 用途 |
|------|------|------|
| dsg-session.md | 项目根目录 | 全局进度、模块列表、当前状态 |
| dsg-task.md | 模块子目录 | 模块进度、步骤状态、执行摘要 |

## 二、dsg-session.md 规范

### 文件位置
- 必须存储在项目根目录

### 核心字段
```markdown
## 当前状态（快速定位）
- 当前阶段：全局设计/模块设计
- 当前模块：[模块名称] 或 null
- 当前步骤：[步骤名称]

## 模块清单
| 序号 | 模块名称 | 优先级 | 状态 | 设计文件夹 |
|------|---------|--------|------|-----------|
```

### 完整格式
```markdown
# 产品设计全局进度

## 当前状态
- 当前阶段：全局设计/模块设计
- 当前模块：[模块名称] 或 null
- 当前步骤：[步骤名称]
- 创建时间：[创建日期]
- 最后更新：[更新日期]
```

## 四、进度更新规则

### 更新时机

| 事件 | 更新文件 | 更新内容 |
|------|---------|---------|
| 开始新项目 | dsg-session.md | 创建文件，初始化基本信息 |
| 全局步骤完成 | dsg-session.md | 更新全局产物状态、当前步骤 |
| 开始新模块 | dsg-session.md | 更新当前模块、模块状态 |
| 模块步骤完成 | dsg-task.md | 更新步骤状态、执行摘要 |
| 模块完成 | dsg-session.md | 更新模块状态、当前模块为null |

### 更新操作

#### 步骤开始时
```markdown
## 当前状态
- 当前步骤：[步骤编号].[步骤名称]
- 步骤状态：进行中
```

#### 步骤完成时
```markdown
## 当前状态
- 当前步骤：[下一步骤编号].[下一步骤名称]
- 步骤状态：未开始

### 步骤X: [步骤名称]
- **状态**: 已完成
- **执行摘要**:
  - [日期] [关键操作]
  - [日期] [用户确认内容]
```

## 五、执行摘要规范

### 摘要内容
- **关键决策**：影响后续步骤的重要决定
- **重要变更**：与原计划的偏差和调整
- **用户确认**：用户明确同意的内容
- **待办事项**：需要后续跟进的问题

### 摘要示例
```markdown
### 步骤1: 功能清单确认
- **状态**: 已完成
- **执行摘要**:
  - 2024-01-16 开始规划功能清单，基于全局功能架构拆解
  - 2024-01-17 用户确认功能清单包含5个页面
  - 2024-01-17 新增"批量导入"功能，用户确认需要Excel模板
  - 关键决策：用户权限页面独立，不与编辑用户合并
  - 用户确认：所有页面都需要支持导出功能
```

## 六、状态转换规则

### 模块状态转换
```
未开始 → 进行中：开始模块设计时
进行中 → 已完成：模块所有必要步骤完成时
```

### 步骤状态转换
```
未开始 → 进行中：开始执行步骤时
进行中 → 已完成：步骤交付物完成且用户确认后
```

### 阶段状态转换
```
全局设计 → 模块设计：全局设计所有产物完成时
模块设计 → 项目完成：所有模块状态为"已完成"时
```

## 七、快速恢复指南

### 恢复操作
1. 读取 dsg-session.md 的"当前状态"部分
2. 根据"当前阶段"确定恢复位置
3. 如在模块设计阶段，读取对应模块的 dsg-task.md
4. 根据"当前步骤"确定具体恢复点
5. 继续执行当前步骤

### 恢复示例
```
dsg-session.md 显示：
- 当前阶段：模块设计
- 当前模块：用户管理
- 当前步骤：原型绘制

恢复操作：
1. 加载 用户管理/dsg-task.md
2. 确认步骤2"原型绘制"的状态
3. 继续执行原型绘制任务
```
