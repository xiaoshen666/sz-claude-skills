## 启动恢复流程（最高优先级）

每次进入时**必须首先执行**以下流程：

### 步骤0：读取进度文件
```
IF 项目根目录存在 devops-session.md THEN
    读取 devops-session.md
    判断当前运维阶段状态
    判断当前部署模块状态
    执行对应的恢复操作
ELSE
    进入"运维环境初始化"步骤
END IF
```

### 恢复操作决策表
| devops-session.md状态 | current_module | current_phase | 恢复操作 |
|-------------------|----------------|--------------|---------|
| 文件不存在 | - | - | 执行步骤0：运维环境初始化 |
| 环境未配置 | - | - | 执行运维环境配置 |
| 环境已配置 | 有值且状态为"进行中" | 构建 | 继续构建流程 |
| 环境已配置 | 有值且状态为"进行中" | 打包 | 继续打包流程 |
| 环境已配置 | 有值且状态为"进行中" | 上传 | 继续上传流程 |
| 环境已配置 | 有值且状态为"进行中" | 部署 | 继续部署流程 |
| 环境已配置 | 无值或状态非"进行中" | - | 选择优先级最高的"未开始"模块 |
| 所有模块已完成 | - | - | 提示用户部署已完成，询问后续操作 |

## 运维流程框架

### 核心原则
- **自动化原则**：全流程自动化，减少人工干预
- **可追溯原则**：每个步骤都有完整的日志记录
- **回滚原则**：任何部署都必须支持快速回滚
- **通知原则**：关键节点通过企微机器人通知相关人员

### 第一阶段：运维环境准备

| 步骤 | 名称 | 详细指南 | 核心交付物 |
|------|------|---------|-----------|
| 0 | 运维环境初始化 | DEVOPS-SETUP.md | devops-session.md、运维环境配置 |
| 1 | 构建环境配置 | BUILD-ENV.md | 构建工具配置、依赖环境配置 |
| 2 | 部署环境配置 | DEPLOY-ENV.md | 目标环境配置、权限配置、网络配置 |
| 3 | 通知系统配置 | NOTIFY-SETUP.md | 企微机器人配置、通知模板配置 |

## 步骤执行规范

### 每个步骤执行前
1. **清理上下文**：移除与当前步骤无关的历史对话
2. **加载指南**：读取对应的 reference 文件
3. **确认前置产物**：检查前置步骤的交付物是否存在

### 每个步骤执行后
1. **更新进度文件**：立即更新 devops-session.md 或 devops-task.md，仅保留关键结论
2. **记录操作日志**：详细记录操作过程、命令执行结果、错误信息（压缩为摘要）
3. **状态监控**：实时监控部署状态，及时发现异常
4. **用户确认**：展示操作结果，获取用户确认后继续
5. **上下文清理**：移除与当前步骤无关的历史对话，仅保留关键结论到进度文件

> **详细上下文清理规范请参考**: [上下文清理规范](references/CONTEXT-CLEANING.md)
