# 前端代码实现检查指南

## 概述

本指南用于检查前端代码实现是否符合前端设计文档的要求，包括组件实现、样式规范、API调用等。检查完成后，结果输出到 `design-docs/testing-validation/` 目录。

## 检查前提

1. ✅ 前端代码已生成完成（选项A已执行）
2. ✅ 前端架构设计文档已存在
3. ✅ UI设计文档已存在
4. ✅ 已确认 `moduleCode`

## 检查步骤

### 步骤1：读取前端设计文档

**查找位置**：
- `design-docs/frontend-architecture/{功能名称}前端架构设计.md`
- `design-docs/ui-design/UI-DESIGN.md`
- `design-docs/ui-design/{功能名称}UI稿.pen`

**提取内容**：
- 组件设计要求
- 样式规范要求
- 交互逻辑要求
- API调用要求
- 状态管理要求

## 检查执行规范

### 检查前准备

1. **清理上下文**：移除与前端代码检查无关的历史对话
2. **加载指南**：读取本检查指南
3. **读取设计文档**：读取前端架构设计和UI设计文档
4. **确认moduleCode**：从 dev-task.md 或 metadata 文件读取

### 检查完成后

1. **生成报告**：将检查报告保存到 `design-docs/testing-validation/{功能名称}-前端代码实现检查报告.md`
2. **更新进度文件**：将检查结论写入 dev-task.md
3. **Git提交**：提交检查报告
4. **上下文清理**：仅保留关键结论到进度文件
5. **后续选择**：询问用户是否继续进行后端检查
