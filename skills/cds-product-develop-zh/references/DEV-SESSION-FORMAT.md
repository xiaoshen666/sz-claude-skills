# dev-session.md 格式规范

## 概述

`dev-session.md` 是**唯一的进度跟踪文件**，用于统一跟踪模块开发进度、记录关键信息和协调前后端开发工作。

## 文件位置

- **位置**：项目根目录或模块目录下
- **文件名**：`dev-session.md`
- **用途**：跨会话的开发进度跟踪和断点续传

## 字段详细说明

### 会话信息

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| 功能名称 | 功能模块的中文名称 | 用户管理、订单管理 | ✅ |
| moduleCode | 模块编码，从 metadata 文件提取 | usermgmt、ordermgmt | ✅ |
| acronym | 模块缩略码，用于数据库表前缀 | USR、ORD | ✅ |
| 会话开始时间 | 会话创建时间 | 2024-01-15 10:30:00 | ✅ |
| 最后更新时间 | 最后一次更新时间 | 2024-01-15 14:20:00 | ✅ |

### 代码目录信息

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| 前端目录 | 前端项目相对路径 | usermgmt-frontend/ | ✅ |
| 后端目录 | 后端自定义代码相对路径 | usermgmt/usermgmt-custom/ | ✅ |

### 开发进度表格

| 字段 | 说明 | 可选值 | 必填 |
|------|------|--------|------|
| 阶段 | 阶段编号 | 1-9 | ✅ |
| 名称 | 阶段名称 | 见上方表格 | ✅ |
| 状态 | 阶段当前状态 | ⬜ 未开始、🔄 进行中、✅ 已完成、⬜ 未选择 | ✅ |
| 完成时间 | 阶段完成时间 | YYYY-MM-DD HH:mm:ss | ❌ |
| 生成文件数 | 该阶段生成的文件数量 | 数字 | ❌ |

### 状态标识说明

- ⬜ **未开始**：该阶段尚未开始执行
- 🔄 **进行中**：该阶段正在执行中
- ✅ **已完成**：该阶段已完成
- ⬜ **未选择**：该阶段未被用户选择执行

## 使用规范

### 创建时机

`dev-session.md` 文件应在以下时机创建：
1. 开始新模块开发时
2. 从设计阶段进入开发阶段时
3. 用户明确指定要开发的模块时

### 更新时机

1. **会话信息更新**：
   - 创建时填写基础信息
   - 每次更新时修改"最后更新时间"

2. **开发方案选择更新**：
   - 用户确认开发方案后立即更新
   - 勾选对应的选项

3. **阶段完成时更新**：
   - 更新对应阶段状态为"✅ 已完成"
   - 记录完成时间
   - 记录生成文件数量
   - 更新"下一阶段"信息

4. **关键决策时更新**：
   - 记录影响后续开发的决策
   - 包括技术选型、架构决策等

5. **Git提交时更新**：
   - 记录重要的Git提交信息
   - 便于代码回退和版本管理

### 清理上下文时

1. 确保 dev-session.md 已更新
2. 确保「下次恢复指南」部分完整
3. 执行 `/clear` 指令
4. 下次从 dev-session.md 的「下次恢复指南」快速恢复

## 字段精简原则

### ✅ 必须保留

1. **功能标识**：功能名称、moduleCode、acronym
2. **进度信息**：已完成阶段、下一阶段
3. **恢复信息**：需要读取的指南、前置产物
4. **目录信息**：前后端代码目录路径
5. **Git记录**：最新提交hash（便于回退）

### ❌ 不需要记录

1. ~~详细的文件完整路径~~（只需记录文件类型和数量）
2. ~~每个阶段的详细执行过程~~（记录在Git提交信息中）
3. ~~编译错误和修复过程~~（临时信息，不需要保留）
4. ~~中间讨论和试错过程~~（清理掉）

## 恢复流程

### 从 dev-session.md 恢复开发

1. **读取会话信息**：确认当前模块和功能
2. **检查开发方案**：确认已选择的选项
3. **查看当前状态**：确定下一阶段
4. **加载指南文件**：读取对应的生成指南
5. **检查前置产物**：确认需要的文件存在
6. **继续开发**：从断点处继续执行

### 恢复检查清单

- [ ] 确认 moduleCode 和 acronym 正确
- [ ] 确认前后端目录存在
- [ ] 确认开发方案选择
- [ ] 确认当前阶段状态
- [ ] 确认前置产物文件存在
- [ ] 确认指南文件可读取
