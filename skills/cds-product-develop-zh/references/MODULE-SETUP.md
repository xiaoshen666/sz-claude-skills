# 模块开发初始化指南

## 概述

本指南定义了模块开发初始化的流程和 `dev-task.md` 文件的格式规范。该文件用于跟踪模块开发进度、记录关键信息和协调前后端开发工作。

## 初始化流程

### 步骤1：创建 dev-task.md

在项目根目录或模块目录下创建 `dev-task.md` 文件，用于记录当前模块的开发任务和进度。

### 步骤2：填充基本信息

根据当前模块的实际情况，填充以下基本信息：
- 模块名称
- moduleCode
- acronym
- 优先级
- 开发状态

### 步骤3：确认前后端目录位置

根据模块代码目录查找规则，确认并记录前后端代码目录位置：
- 前端目录：`{moduleCode}-frontend/`
- 后端目录：`{moduleCode}-custom/`

### 步骤4：初始化任务清单

根据开发流程框架，初始化任务清单，包含所有开发步骤。

## 字段说明

### 基本信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 模块名称 | 功能模块的中文名称 | 用户管理、订单管理 |
| moduleCode | 模块编码，从 metadata 文件提取 | usermgmt、ordermgmt |
| acronym | 模块缩略码，用于数据库表前缀 | USR、ORD |
| 优先级 | 模块开发优先级 | 高、中、低 |
| 开发状态 | 当前模块开发状态 | 未开始、进行中、已完成 |

### 代码目录信息

| 字段 | 说明 | 示例 |
|------|------|------|
| frontend目录路径 | 前端项目相对路径 | usermgmt-frontend/ |
| custom目录路径 | 后端自定义代码相对路径 | usermgmt/usermgmt-custom/ |
| 目录状态 | 目录是已存在还是新创建 | 已存在、新创建 |

## 使用规范

### 创建时机

`dev-task.md` 文件应在以下时机创建：
1. 开始新模块开发时
2. 从 dev-session.md 切换到具体模块开发时
3. 用户明确指定要开发的模块时

### 更新规则

1. **实时更新**：每完成一个步骤，立即更新对应的任务状态和时间
2. **状态同步**：确保 dev-task.md 与 dev-session.md 的状态保持一致
3. **信息完整**：所有必填字段必须填写完整，不得留空

### 状态标记说明

- ⬜ 未开始：任务尚未开始
- 🔄 进行中：任务正在进行中
- ✅ 已完成：任务已完成
- ❌ 已取消：任务已取消（需说明原因）

## 与 dev-session.md 的关系

### 职责区分

- **dev-session.md**：记录整个开发会话的全局状态和进度
- **dev-task.md**：记录当前具体模块的开发任务和详细信息

### 同步机制

1. 当开始新模块开发时，从 dev-session.md 读取全局配置信息
2. 在 dev-task.md 中记录模块级别的详细信息
3. 模块开发完成后，将结果同步回 dev-session.md

### 信息流转

```
dev-session.md (全局配置)
    ↓
dev-task.md (模块任务)
    ↓
执行开发步骤
    ↓
更新 dev-task.md
    ↓
同步到 dev-session.md
```
