# 上下文清理规范

## 概述

本规范定义了CDS开发阶段执行时的上下文清理策略，目的是减少Token消耗、提高相关信息密度、避免上下文窗口溢出。

## 清理原则

1. **保留关键结论**：将核心结果、关键决策、用户确认写入进度文件
2. **清理中间过程**：移除调试信息、试错过程、临时文件路径
3. **压缩历史信息**：将详细执行过程压缩为执行摘要
4. **按需加载**：详细内容存放在references目录，仅在需要时读取

## 进度文件格式

**进度文件**：dev-session.md、dev-task.md

**保留内容**：
- 当前开发阶段和步骤
- 前后端代码目录路径
- moduleCode和acronym
- Git提交记录
- 关键开发决策

**清理内容**：
- 代码生成过程
- 编译错误和修复过程
- 调试信息
- 中间讨论和试错过程

## /clear 指令使用规则

### 使用时机
1. 阶段完成后
2. 对话历史超过50轮
3. 切换到新模块时
4. 用户明确要求时

### 清理前准备
1. ✅ 所有关键结论已写入进度文件
2. ✅ 所有产物已保存到正确位置
3. ✅ 所有用户确认已记录
4. ✅ 下一步操作已明确

## 最佳实践

1. **及时清理**：每个步骤完成后立即清理
2. **完整记录**：清理前确保所有关键信息已保存
3. **按需加载**：详细内容放在references目录
4. **用户透明**：清理过程对用户透明，只看到关键结论
