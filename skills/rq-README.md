# 快速开发流程技能集 (rq-系列)

## 概述
这是一套专为小型项目设计的快速开发流程技能集，包含三个核心技能，实现从需求到部署的完整流程。

## 技能列表

### 1. rq-product-designer (产品设计师)
- **功能**：快速生成产品设计文档
- **输入**：项目需求描述
- **输出**：`docs/{项目名称}-产品设计文档.md`
- **下一步**：自动创建UI设计任务，提示调用UI设计师

### 2. rq-ui-designer (UI设计师)
- **功能**：快速生成UI设计和规范
- **输入**：产品设计文档
- **输出**：
  - `docs/{项目名称}-UI设计文档.md`
  - `ui/{项目名称}.pen` (pencil UI稿)
- **下一步**：自动创建开发任务，提示调用开发者

### 3. rq-developer (开发者)
- **功能**：快速实现代码和部署
- **输入**：产品设计文档 + UI设计文档
- **输出**：
  - `docs/{项目名称}-开发计划.md`
  - `src/frontend/` (前端代码)
  - `src/backend/` (后端代码)
  - `docs/{项目名称}-部署文档.md`
  - `docs/{项目名称}-项目完成报告.md`

## 使用流程

### 步骤1：启动产品设计
```
/rq-product-designer
```
1. 提供项目名称
2. 提供需求描述
3. 技能自动生成产品设计文档
4. 自动创建UI设计任务

### 步骤2：进行UI设计
```
/rq-ui-designer
```
1. 技能自动读取产品设计文档
2. 生成UI设计文档
3. 使用pencil MCP生成UI稿
4. 自动创建开发任务

### 步骤3：进行开发实现
```
/rq-developer
```
1. 技能自动读取所有设计文档
2. 制定开发计划
3. 生成前后端代码
4. 生成部署文档
5. 输出项目完成报告

## 文件夹结构

```
项目根目录/
├── docs/                    # 所有文档
│   ├── {项目名称}-产品设计文档.md
│   ├── {项目名称}-UI设计文档.md
│   ├── {项目名称}-开发计划.md
│   ├── {项目名称}-部署文档.md
│   └── {项目名称}-项目完成报告.md
├── ui/                     # UI设计文件
│   └── {项目名称}.pen      # pencil UI稿
└── src/                    # 源代码
    ├── frontend/          # 前端代码
    └── backend/           # 后端代码
```

## 特点

### 1. 快速高效
- 每个技能专注于一个环节
- 文档简洁实用，避免过度详细
- 自动衔接流程，减少人工干预

### 2. 专业分工
- 产品设计师：需求分析和产品设计
- UI设计师：界面设计和规范制定
- 开发者：代码实现和部署

### 3. 自动化流程
- 自动创建任务文件
- 自动提示下一步操作
- 自动检查输入完整性

### 4. 标准化输出
- 统一的文档格式
- 标准化的文件夹结构
- 完整的项目交付物

## 适用场景

- ✅ 内部工具开发
- ✅ 原型验证项目
- ✅ 快速概念验证
- ✅ 个人项目开发

## 技术栈建议

### 前端推荐
- React + Ant Design
- Vue + Element UI
- 根据项目复杂度选择

### 后端推荐
- Node.js + Express
- Python + Flask
- Java + Spring Boot (小型项目)

### 数据库推荐
- SQLite (简单项目)
- PostgreSQL (需要关系型数据库)
- MongoDB (需要文档型数据库)

## 注意事项

1. **保持简洁**：每个文档只包含必要信息
2. **快速迭代**：不要过度设计，快速实现
3. **实用为主**：选择成熟、易用的技术
4. **文档完整**：确保所有必要文档都已生成
5. **流程顺畅**：按照顺序调用技能

## 开始使用

1. 确保技能已安装到skills目录
2. 在项目目录中启动第一个技能
3. 按照提示完成整个流程

## 技能内部专业程度

每个技能都基于现有大型技能集的专业要求进行了简化，确保：
- **专业性不降低**：核心专业能力保留
- **复杂度降低**：去除大型项目的复杂流程
- **实用性增强**：专注于小型项目的实际需求
- **速度提升**：减少文档数量，加快流程速度
