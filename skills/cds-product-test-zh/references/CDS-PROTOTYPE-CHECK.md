# CDS原型检查监督指南

## 检查目标
在原型绘制过程中进行实时质量监督，确保每个页面的原型设计符合CDS设计规范、功能需求和技术要求。

## 前置验证
执行前必须验证：
- 模块功能清单已确认
- CDS设计组件库已准备（.lib.pen、design-libraries-readme.md）
- 当前页面原型已完成绘制

## 检查清单

### 1. CDS设计规范符合性检查

#### 1.1 CDS组件使用检查
- [ ] 是否使用CDS标准组件库中的组件
- [ ] 组件使用是否符合CDS规范
- [ ] 组件层级和样式是否正确
- [ ] 是否有自定义组件需要特殊说明

#### 1.2 CDS布局规范检查
- [ ] 页面布局是否符合CDS栅格系统
- [ ] 间距和对齐是否遵循CDS标准
- [ ] 响应式设计是否考虑
- [ ] 页面结构是否符合CDS框架

#### 1.3 CDS视觉规范检查
- [ ] 颜色使用是否符合CDS色彩体系
- [ ] 字体使用是否符合CDS字体规范
- [ ] 图标使用是否符合CDS图标库
- [ ] 动效设计是否符合CDS交互规范

### 2. 功能完整性检查

#### 2.1 功能覆盖检查
- [ ] 页面功能是否完整覆盖功能清单
- [ ] 核心操作是否都有对应界面
- [ ] 辅助功能是否考虑周全
- [ ] 异常情况处理界面是否设计

#### 2.2 交互流程检查
- [ ] 用户操作流程是否清晰
- [ ] 页面跳转逻辑是否合理
- [ ] 状态转换是否明确
- [ ] 反馈机制是否完善

### 3. CDS平台兼容性检查

#### 3.1 CDS组件兼容性
- [ ] 使用的组件是否在CDS平台可用
- [ ] 组件属性设置是否符合CDS要求
- [ ] 组件交互方式是否与CDS一致
- [ ] 是否有CDS不支持的组件使用

#### 3.2 CDS数据交互检查
- [ ] 数据展示格式是否符合CDS标准
- [ ] 数据输入验证是否合理
- [ ] 数据表格是否符合CDS规范
- [ ] 图表组件是否使用CDS标准

### 4. 用户体验检查

#### 4.1 操作便捷性
- [ ] 常用操作是否便捷
- [ ] 复杂操作是否有引导
- [ ] 错误操作是否有预防
- [ ] 操作反馈是否及时

#### 4.2 信息架构
- [ ] 信息层次是否清晰
- [ ] 重要信息是否突出
- [ ] 信息密度是否合理
- [ ] 辅助信息是否恰当

## 检查执行流程

### 步骤1：页面原型检查
```
IF 当前页面原型已完成 THEN
    执行页面原型检查
ELSE
    提示用户：请先完成当前页面原型绘制
END IF
```

### 步骤2：逐项检查
按照检查清单逐项检查，记录发现的问题

### 步骤3：生成检查报告
生成详细的页面检查报告

## 问题处理原则

### 问题分类处理
- **严重问题**：必须修复，影响功能或CDS兼容性
- **中等问题**：建议修复，影响用户体验
- **轻微问题**：可酌情处理，主要是样式优化

### CDS相关问题优先级
- **CDS兼容性**：必须确保，影响平台集成
- **CDS规范符合性**：优先处理，确保一致性
- **功能完整性**：核心要求，必须满足
- **用户体验优化**：在时间允许情况下优化

## 后续处理

### 检查通过
- 记录检查结果
- 继续下一个页面检查
- 更新检查进度

### 检查不通过
- 提供详细的问题说明
- 给出具体的修改建议
- 等待设计师修改后重新检查
- 确保问题闭环

## 与设计师协作
- 提供清晰的问题描述和修改建议
- 解释CDS规范要求
- 协助理解CDS平台限制
- 确保修改方向正确

## 检查频率建议
- 每完成一个页面立即检查
- 避免批量检查导致问题积累
- 确保问题及时发现和解决
- 提高原型设计质量
