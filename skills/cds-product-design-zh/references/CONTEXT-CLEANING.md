# 上下文清理规范

## 概述

本规范定义了CDS Skills在执行长链路任务时的上下文清理策略，目的是：
- 减少Token消耗
- 提高相关信息密度
- 避免上下文窗口溢出
- 保持对话历史清洁

## 清理原则

### 核心原则

1. **保留关键结论**：将核心结果、关键决策、用户确认写入进度文件
2. **清理中间过程**：移除调试信息、试错过程、临时文件路径
3. **压缩历史信息**：将详细执行过程压缩为执行摘要
4. **按需加载**：详细内容存放在references目录，仅在需要时读取

## 清理时机

### 每个步骤执行完成后

在每个步骤执行完成后，必须执行上下文清理：

```markdown
### 上下文清理规则

每个步骤执行完成后，必须执行上下文清理：
1. **保留关键结论**：将核心结果、关键决策、用户确认写入进度文件
2. **清理中间过程**：移除调试信息、试错过程、临时文件路径
3. **压缩历史信息**：将详细执行过程压缩为执行摘要
4. **使用/clear指令**：在适当时机使用系统指令清理对话上下文
```

### 阶段切换时

当从一个阶段切换到另一个阶段时：

1. **保存当前阶段结论**：将当前阶段的关键结论写入进度文件
2. **清理阶段相关上下文**：移除与下一阶段无关的历史对话
3. **加载下一阶段指南**：按需读取下一阶段所需的references文件
4. **初始化新阶段**：按照新阶段的初始化流程执行

## 进度文件格式

### 关键结论格式

所有进度文件（dsg-session.md、dsg-task.md）应遵循以下格式：

```markdown
## 关键结论

### 当前状态
- **阶段**：{阶段名称}
- **步骤**：{步骤名称}
- **状态**：{未开始/进行中/已完成}

### 核心结果
- {结果1}
- {结果2}
- {结果3}

### 关键决策
- {决策1}：{决策内容}
- {决策2}：{决策内容}

### 用户确认
- {确认项1}：{确认结果}
- {确认项2}：{确认结果}

### 执行摘要
- {步骤}：{简要描述}（{时间}）
- {步骤}：{简要描述}（{时间}）
```

### 压缩规则

1. **详细执行过程** → 压缩为一行执行摘要
2. **调试信息** → 完全移除，不保留
3. **试错过程** → 仅保留最终方案
4. **临时文件路径** → 仅保留最终产物路径
5. **代码片段** → 不保留在进度文件，仅保留文件路径

## /clear 指令使用规则

### 使用时机

在以下情况下建议使用 `/clear` 或系统指令清理上下文：

1. **阶段完成后**：一个完整阶段的所有步骤都已完成
2. **上下文过长时**：对话历史超过50轮
3. **切换到新模块时**：开始处理新的功能模块
4. **用户明确要求时**：用户主动要求清理上下文

### 清理前准备

在执行 `/clear` 之前，必须确保：

1. ✅ 所有关键结论已写入进度文件
2. ✅ 所有产物已保存到正确位置
3. ✅ 所有用户确认已记录
4. ✅ 下一步操作已明确

### 清理后恢复

清理上下文后，通过以下方式恢复：

1. 读取进度文件（dsg-session.md、dsg-task.md等）
2. 根据进度文件判断当前状态
3. 加载当前步骤所需的references文件
4. 继续执行下一步操作

## 清理策略

**进度文件**：dsg-session.md、dsg-task.md

**保留内容**：
- 当前设计阶段和步骤
- 设计产物路径
- moduleCode和acronym
- 用户确认的设计流程模式
- 关键设计决策

**清理内容**：
- 详细设计过程
- UI稿生成过程
- 架构设计草稿
- 中间讨论和试错过程

## Token优化效果

通过上下文清理机制，预期达到以下效果：

| 指标 | 清理前 | 清理后 | 优化比例 |
|------|--------|--------|---------|
| 上下文Token数 | ~50,000 | ~10,000 | 80% |
| 相关信息密度 | ~10% | ~80% | 8倍 |
| 响应速度 | ~2s | ~0.5s | 4倍 |
| 推理质量 | 中等 | 高 | 显著提升 |

## 最佳实践

1. **及时清理**：每个步骤完成后立即清理，不要累积
2. **完整记录**：清理前确保所有关键信息已保存到进度文件
3. **按需加载**：详细内容放在references目录，不要全部加载到上下文
4. **定期归档**：对于长期项目，定期将旧进度文件归档
5. **用户透明**：清理过程对用户透明，用户只看到关键结论
