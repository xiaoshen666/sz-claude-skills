# 测试用例生成指南

## 核心目标
基于PRD内容编写详细的测试用例文档，为后续测试执行提供依据。

## 前置产物
- 模块PRD.md
- 模块功能清单.md
- 模块数据架构.md
- 模块业务流程.md

## 测试用例结构
```markdown
# 测试用例：[功能模块]-[测试场景]
- **测试ID**：TC-[模块]-[序号]
- **测试目的**：验证[具体功能]是否符合PRD要求
- **前置条件**：[测试前需要满足的条件]
- **测试步骤**：
  1. [步骤1]
  2. [步骤2]
- **预期结果**：[期望的输出结果]
```

## 测试用例分类

### 1. 功能测试用例
验证功能是否按PRD要求正常工作

### 2. 边界测试用例
测试输入的边界条件：
- 空值输入
- 最大长度输入
- 最小值/最大值
- 特殊字符

### 3. 异常测试用例
测试异常情况下的系统表现：
- 网络异常
- 权限不足
- 数据冲突
- 并发操作

### 4. 业务流程测试用例
基于业务流程图的端到端测试

## 测试用例编写流程

### 步骤1：读取PRD
分析PRD中的功能需求和验收标准

### 步骤2：识别测试场景
基于功能模块和业务流程识别测试场景

### 步骤3：编写测试用例
按照测试用例结构编写详细用例

### 步骤4：整理输出
输出完整的测试用例文档

## 测试重点
- **核心功能**：优先覆盖PRD中的核心功能
- **业务流程**：覆盖完整的业务流程
- **数据验证**：确保数据处理符合PRD要求
- **界面交互**：覆盖用户界面交互场景

## 测试用例示例

### 功能测试用例
```markdown
### TC-001：正常登录
- **测试目的**：验证用户使用正确的用户名和密码能够正常登录
- **前置条件**：用户已注册，系统运行正常
- **测试步骤**：
  1. 打开登录页面
  2. 输入正确的用户名：test@example.com
  3. 输入正确的密码：password123
  4. 点击登录按钮
- **预期结果**：成功登录并跳转到首页
```

### 边界测试用例
```markdown
### TC-002：用户名最大长度
- **测试目的**：验证用户名输入最大长度限制
- **前置条件**：系统运行正常
- **测试步骤**：
  1. 打开注册页面
  2. 输入超过最大长度（如100字符）的用户名
  3. 点击注册按钮
- **预期结果**：显示错误提示"用户名不能超过50个字符"
```

### 异常测试用例
```markdown
### TC-003：网络异常处理
- **测试目的**：验证网络异常时的系统表现
- **前置条件**：用户已登录，系统运行正常
- **测试步骤**：
  1. 断开网络连接
  2. 执行需要网络的操作
- **预期结果**：显示友好的网络异常提示，不导致系统崩溃
```

## 输出文件
- 文件名：`测试用例.md`
- 位置：模块子目录
- 包含内容：
  - 测试用例概述
  - 功能测试用例（按模块分类）
  - 业务流程测试用例
  - 边界测试用例
  - 异常测试用例
  - 测试数据准备清单
