# 全局设计检查指南

## 检查目标
验证全局设计阶段（需求分析、功能架构、组件库准备）是否按要求完成，是否存在遗漏。

## 前置验证
执行前必须验证：
- dsg-session.md 全局设计阶段状态为"已完成"
- 01-需求分析.md 存在
- 02-功能架构.md 存在
- .lib.pen 存在
- design-libraries-readme.md 存在

## 检查清单

### 1. 需求分析检查

#### 1.1 文档完整性
- [ ] 需求描述是否清晰完整
- [ ] 产品目标是否明确
- [ ] 用户群体是否定义

#### 1.2 端及角色表检查
- [ ] 是否列出所有端（Web/App/小程序等）
- [ ] 是否定义所有用户角色
- [ ] 角色权限是否初步描述

#### 1.3 需求清单表检查
- [ ] 是否列出所有功能需求
- [ ] 需求优先级是否标注
- [ ] 需求描述是否清晰

### 2. 功能架构检查

#### 2.1 文档完整性
- [ ] 功能模块划分是否清晰
- [ ] 模块层级是否合理
- [ ] 模块边界是否明确

#### 2.2 核心功能模块表检查
- [ ] 是否包含所有模块
- [ ] 每个模块是否有子模块/页面/操作
- [ ] 操作是否与角色关联
- [ ] 模块优先级是否标注

#### 2.3 模块优先级计划检查
- [ ] 是否有明确的开发顺序
- [ ] 优先级排序是否合理
- [ ] 模块依赖关系是否考虑

### 3. 设计组件库检查

#### 3.1 组件库文件检查
- [ ] .lib.pen 文件是否存在
- [ ] 是否包含基础组件（按钮、输入框、卡片等）
- [ ] 是否包含布局组件（导航、侧边栏、表格等）
- [ ] 组件样式是否统一

#### 3.2 设计风格说明检查
- [ ] design-libraries-readme.md 是否存在
- [ ] 设计风格方向是否明确
- [ ] 主色调和辅助色是否定义
- [ ] 字体规范是否完整
- [ ] 间距规范是否定义

### 4. 一致性检查

#### 4.1 需求与架构一致性
- [ ] 功能架构是否覆盖所有需求
- [ ] 是否有需求遗漏
- [ ] 是否有超出需求的功能

#### 4.2 组件库与产品定位一致性
- [ ] 组件库风格是否符合产品定位
- [ ] 组件库是否满足功能需求

## 后续处理
- 如检查通过：product-manager 读取报告后更新 dsg-session.md 状态并继续

## 与 product-manager 协作
- tester 只负责执行检查和输出报告
- tester 不修改任何进度文件
- product-manager 负责读取报告、调整修改、更新进度
