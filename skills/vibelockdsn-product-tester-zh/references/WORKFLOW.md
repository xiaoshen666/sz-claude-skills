# 测试验收工作流程

## 步骤衔接规则

### 设计阶段检查衔接
```
全局设计完成 → 全局设计检查
    触发条件：dsg-session.md 全局设计状态为"已完成"
    前置验证：检查产物文件存在
    输出：全局设计检查报告.md
    
模块设计完成 → 模块设计检查
    触发条件：dsg-task.md 模块设计状态为"设计完成待检查"
    前置验证：检查产物文件存在
    输出：模块设计检查报告.md
    
PRD/测试用例完成 → PRD测试用例检查
    触发条件：dsg-task.md PRD和测试用例状态为"已完成"
    前置验证：检查产物文件存在
    输出：PRD测试用例检查报告.md
```

### 开发阶段测试衔接
```
模块开发完成 → 模块测试
    触发条件：dev-task.md 模块开发状态为"开发完成"
    前置验证：检查测试用例存在
    输出：测试报告.md
    
所有模块开发完成 → 集成测试
    触发条件：dev-session.md 所有模块状态为"已完成"
    前置验证：检查所有模块测试通过
    输出：集成测试报告.md
    
集成测试通过 → 项目验收
    触发条件：集成测试报告结论为"通过"
    前置验证：检查所有产物完整
    输出：验收报告.md
```

## 上下文清理规则

### 进入新检查任务前
1. 清理上一个检查任务的详细报告内容
2. 保留当前任务的前置产物路径
3. 只加载当前检查所需的 reference 文件

### 检查完成后
1. 输出检查报告
2. 清理检查过程中的临时数据
3. 等待 product-manager 读取报告

## 异常处理

### 前置验证失败
```
IF 产物文件不存在 THEN
    提示用户缺少必要产物
    列出缺失文件清单
    EXIT（等待 product-manager 完成产物）
END IF
```

### 检查发现问题
```
IF 检查结论为"不通过" THEN
    输出问题清单
    等待 product-manager 修复
    修复后重新执行检查
END IF
```

### 用户中断
```
IF 用户要求中断检查 THEN
    保存当前检查进度
    记录已完成的检查项
    EXIT
END IF
```

## 与 product-manager 协作流程

### 检查请求流程
1. product-manager 完成设计产物
2. product-manager 调用 tester 执行检查
3. tester 执行前置验证
4. tester 执行检查并输出报告
5. product-manager 读取报告
6. product-manager 根据报告调整或继续

### 问题反馈流程
1. tester 发现问题，记录到检查报告
2. product-manager 读取报告
3. product-manager 根据问题清单修改
4. product-manager 确认修改完成
5. product-manager 调用 tester 重新检查
6. 循环直到检查通过

### 进度文件更新
- tester 只负责输出检查报告
- product-manager 负责更新 dsg-session.md 和 dsg-task.md 状态
- tester 不修改任何进度文件
