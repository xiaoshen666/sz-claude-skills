# 模块测试执行指南

## 测试原则
- 必须以测试用例为主要依据
- 测试不通过必须修复后重测，不得跳过失败项
- 最终输出完整测试报告

## 前置产物
- 模块测试用例.md
- 模块功能清单.md
- 模块业务流程.md
- 模块PRD.md

## 测试执行流程

### 1. 准备测试数据
- 按测试数据清单准备数据
- 确保数据库中有足够的测试数据覆盖各场景

### 2. 单元测试
- 后端：测试每个 Service 方法和 API 接口
- 前端：测试核心组件的渲染和交互逻辑
- 数据库：验证数据约束和关系

**执行命令参考：**
- Vue3/React：`npm run test`（Vitest / Jest）
- Python FastAPI：`pytest tests/`
- Node.js：`npm test`

### 3. 集成测试
- 测试 API 接口的完整请求-响应链路
- 验证前后端数据格式一致性
- 测试认证授权流程（如有）

### 4. 端到端测试
- 按测试用例中的用户场景逐条执行
- 覆盖核心业务流程（登录→操作→退出等）
- 验证页面跳转、数据展示、操作反馈

### 5. 边界与异常测试
- 空值、超长输入、非法字符
- 网络异常、接口超时
- 权限越界访问

## 补充测试用例
识别测试用例未覆盖的场景，补充：
- 新发现的边界情况
- 开发过程中新增的功能点
- 性能相关测试（响应时间、并发）

## 输出测试报告

测试完成后，在模块子目录输出以下格式：

```markdown
# 测试报告

## 测试概览
- 测试日期：
- 测试版本：
- 总用例数：
- 通过：  / 失败：  / 跳过：
- 覆盖率：

## 单元测试结果
| 模块 | 用例描述 | 结果 | 备注 |
|------|---------|------|------|
| ...  | ...     | ✅/❌ | ...  |

## 集成测试结果
| 接口 | 场景 | 结果 | 备注 |
|------|------|------|------|

## 端到端测试结果
| 用户场景 | 步骤 | 结果 | 备注 |
|---------|------|------|------|

## 失败用例分析
（列出所有失败用例及修复方案）

## 修复记录
（记录测试过程中发现并修复的问题）

## 结论
（整体质量评估，是否满足交付标准）
```

## 输出文件
- 文件名：`测试报告.md`
- 位置：模块子目录

## 修复与回归
- 对所有失败用例进行修复
- 修复后重新执行相关测试用例
- 直至所有用例通过，方可进入下一步骤

## Git提交
测试通过后：
```bash
git add .
git commit -m "[模块] 测试执行完成

- 执行单元测试、集成测试、端到端测试
- 输出测试报告
- 修复发现的问题"
git tag v[X].3-tested
```
