# PRD和测试用例检查指南

## 检查目标
验证PRD和测试用例是否涵盖功能清单中的全部功能，确保文档完整性和一致性。

## 前置验证
执行前必须验证：
- dsg-task.md PRD状态为"已完成"
- dsg-task.md 测试用例状态为"已完成"
- [模块名]/PRD.md 存在
- [模块名]/测试用例.md 存在
- [模块名]/功能清单.md 存在

## 检查清单

### 1. PRD覆盖性检查

#### 1.1 功能覆盖
- [ ] PRD是否涵盖功能清单中的所有页面
- [ ] PRD是否涵盖功能清单中的所有操作
- [ ] PRD是否涵盖功能清单中的所有角色

#### 1.2 页面规范覆盖
- [ ] 每个页面是否有页面规范
- [ ] 页面规范是否包含字段说明
- [ ] 页面规范是否包含操作说明
- [ ] 页面规范是否包含交互说明

#### 1.3 技术实现覆盖
- [ ] 是否有技术实现指南
- [ ] 技术实现是否涵盖所有功能
- [ ] 是否有性能要求
- [ ] 是否有安全要求

#### 1.4 验收标准覆盖
- [ ] 是否有验收清单
- [ ] 验收标准是否明确
- [ ] 验收标准是否可测试

### 2. 测试用例覆盖性检查

#### 2.1 功能测试覆盖
- [ ] 测试用例是否涵盖所有页面
- [ ] 测试用例是否涵盖所有操作
- [ ] 测试用例是否涵盖所有角色场景

#### 2.2 测试类型覆盖
- [ ] 是否有功能测试用例
- [ ] 是否有边界测试用例
- [ ] 是否有异常测试用例
- [ ] 是否有业务流程测试用例

#### 2.3 测试用例质量
- [ ] 测试步骤是否清晰
- [ ] 预期结果是否明确
- [ ] 前置条件是否完整
- [ ] 测试数据是否准备

### 3. 一致性检查

#### 3.1 PRD与功能清单一致性
- [ ] PRD中的页面与功能清单是否一致
- [ ] PRD中的操作与功能清单是否一致
- [ ] PRD中的角色与功能清单是否一致

#### 3.2 测试用例与PRD一致性
- [ ] 测试用例是否覆盖PRD中的所有功能
- [ ] 测试用例的预期结果是否与PRD一致
- [ ] 测试用例的验收标准是否与PRD一致

#### 3.3 测试用例与功能清单一致性
- [ ] 测试用例是否覆盖功能清单中的所有功能
- [ ] 测试用例的操作是否与功能清单一致

### 4. 文档质量检查

#### 4.1 PRD质量
- [ ] 核心摘要是否清晰
- [ ] 页面规范是否完整
- [ ] 技术实现是否可行
- [ ] 验收清单是否完整

#### 4.2 测试用例质量
- [ ] 测试用例编号是否规范
- [ ] 测试步骤是否可执行
- [ ] 预期结果是否可验证
- [ ] 测试数据是否可准备
