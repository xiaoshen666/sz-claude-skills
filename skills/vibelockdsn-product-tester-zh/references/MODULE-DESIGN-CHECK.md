# 模块设计检查指南

## 检查目标
验证模块设计是否符合风格文档要求、是否使用了组件库、是否完成了全部页面设计。

## 前置验证
执行前必须验证：
- dsg-task.md 模块设计状态为"设计完成待检查"
- [模块名]/功能清单.md 存在
- [模块名]/数据架构.md 存在
- [模块名]/业务流程.md 存在
- [模块名]/原型.pen 存在
- [模块名]/设计规范.md 存在

## 检查清单

### 1. 设计规范符合性检查

#### 1.1 风格文档一致性
- [ ] 设计规范是否与 design-libraries-readme.md 风格一致
- [ ] 主色调是否与全局风格一致
- [ ] 字体规范是否与全局规范一致
- [ ] 间距规范是否与全局规范一致

#### 1.2 设计规范完整性
- [ ] 是否包含设计风格方向描述
- [ ] 是否包含字体规范
- [ ] 是否包含颜色规范
- [ ] 是否包含布局规范
- [ ] 是否包含组件规范

### 2. 组件库使用检查

#### 2.1 组件库引用
- [ ] 原型是否导入了 .lib.pen 组件库
- [ ] 是否优先使用组件库中的组件
- [ ] 自定义组件是否有必要

#### 2.2 组件使用一致性
- [ ] 相同功能是否使用相同组件
- [ ] 组件样式是否统一
- [ ] 组件交互是否一致

### 3. 页面完整性检查

#### 3.1 功能清单对照
- [ ] 功能清单中的每个页面是否都有原型
- [ ] 页面名称是否与功能清单一致
- [ ] 页面数量是否与功能清单匹配

#### 3.2 页面内容检查
- [ ] 每个页面是否包含必要的元素
- [ ] 页面布局是否合理
- [ ] 页面交互是否清晰

### 4. 数据架构一致性检查

#### 4.1 字段一致性
- [ ] 原型中的字段是否来自数据架构
- [ ] 字段名称是否与数据架构一致
- [ ] 字段类型是否匹配

#### 4.2 数据关系一致性
- [ ] 数据关系是否正确体现
- [ ] 主外键关系是否清晰

### 5. 业务流程一致性检查

#### 5.1 流程完整性
- [ ] 原型导航是否符合业务流程
- [ ] 操作流程是否与流程图一致
- [ ] 状态转换是否正确实现

#### 5.2 异常处理
- [ ] 是否考虑异常情况
- [ ] 错误提示是否设计

## 后续处理
- 如检查通过：product-manager 读取报告后更新 dsg-task.md 状态并继续
- 如检查不通过：product-manager 根据问题清单修复后重新调用检查

## 与 product-manager 协作
- tester 只负责执行检查和输出报告
- tester 不修改任何进度文件
- product-manager 负责读取报告、调整修改、更新进度
