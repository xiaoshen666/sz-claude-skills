# 读取产品设计产物

## 前置条件
- 用户已完成 `vibelockdsn-product-manager-zh` 的产品设计
- 项目根目录下存在 `dsg-session.md`（设计阶段全局进度文件）
- 项目根目录下存在设计文件夹（含所有 `.md` 和 `.pen` 文件）

## 步骤

### 0. 启动恢复流程

每次进入时执行以下流程：

#### 步骤0.1：读取dsg-session.md（设计进度）
- 读取项目根目录下的 `dsg-session.md`
- 获取所有功能模块列表和优先级
- 获取每个模块的设计状态（未开始/进行中/已完成）
- 确定哪些模块已完成设计，可以开始开发

#### 步骤0.2：读取dev-session.md（开发进度）
- 读取项目根目录下的 `dev-session.md`
- 如不存在，创建新的 `dev-session.md` 文件
- 获取当前正在开发的模块（`current_module` 字段）
- 获取每个模块的开发状态（未开始/进行中/已完成）

#### 步骤0.3：确定任务恢复点
根据 `dsg-session.md` 和 `dev-session.md` 中的信息确定任务恢复点：

| 设计阶段状态 | 开发阶段状态 | 恢复操作 |
|-------------|-------------|---------|
| 设计未完成 | - | 提示用户先完成设计阶段 |
| 设计已完成 | 开发未开始 | 从第一个已设计模块开始开发 |
| 设计已完成 | 存在"进行中"模块 | 加载该模块，读取设计产物继续 |
| 设计已完成 | 无"进行中"模块 | 选择下一个已完成设计的模块 |
| 所有模块已开发 | - | 进入集成与交付阶段 |

#### 步骤0.4：加载模块目录
- 如需继续某个模块的开发：
  - 从 `dsg-session.md` 获取模块的设计文件夹路径
  - 切换工作目录到模块文件夹
  - 准备读取模块的设计产物

#### 步骤0.5：任务恢复提示
向用户展示当前进度：
```
## 任务恢复提示

### 当前项目状态
- 产品名称：企业管理系统
- 设计进度：已完成 3/5 模块
- 开发进度：已完成 1/3 模块
- 当前模块：用户管理（P0）
- 模块状态：进行中

### 当前模块进度
- 模块名称：用户管理
- 已完成：后端开发
- 当前步骤：前端开发

### 下一步操作
继续执行"前端开发"步骤，是否继续？
1. 继续当前任务
2. 查看详细进度
3. 切换到其他模块
```

## 模块化开发流程

### 开发顺序
1. 从 `dsg-session.md` 获取模块设计完成顺序
2. 按设计完成顺序开发各模块（已完成设计的模块）
3. 每个模块开发完成后，更新 `dev-session.md` 中的模块状态

### 模块开发流程
1. 读取当前模块的 `dsg-task.md` 了解设计进度
2. 创建模块的 `dev-task.md` 记录开发进度
3. 读取当前模块的功能清单、原型、设计规范等产物
4. 执行开发任务
5. 更新模块 `dev-task.md` 记录开发进度
