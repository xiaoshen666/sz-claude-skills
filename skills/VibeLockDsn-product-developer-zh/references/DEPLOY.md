# 构建部署指南

## 目标
配置生产环境构建，生成部署配置和CI/CD配置，输出部署文档和运维指南。

## 一、读取设计产物

运维工程师必须读取以下设计产物：

| 产物 | 文件 | 用途 |
|------|------|------|
| 模块PRD | [模块名]/PRD.md | 获取技术栈和环境要求 |
| 模块数据架构 | [模块名]/数据架构.md | 了解数据库配置需求 |
| 模块设计规范 | [模块名]/设计规范.md | 了解前端构建要求 |

## 二、步骤

### 1. 配置生产环境构建
- 设置前端生产构建配置
- 配置后端生产环境构建
- 优化构建输出和性能

### 2. 性能优化和代码压缩
- 启用代码压缩和混淆
- 配置资源优化（图片、CSS、JS）
- 设置缓存策略

### 3. 生成部署配置
- 创建环境配置文件
- 配置数据库连接参数
- 设置应用服务器配置
- **输出数据库本地连接信息**：地址、端口、用户名、密码等连接参数

### 4. 准备CI/CD配置
- 配置持续集成流程
- 设置自动化测试步骤
- 配置自动化部署流程

### 5. 设置监控和日志系统
- 配置应用日志记录
- 设置性能监控
- 配置告警机制

### 6. 输出部署文档
在 `docs/` 下创建：
- `DEPLOY.md` — 部署指南
- `OPERATIONS.md` — 运维手册

### 7. 输出数据库连接信息
- **数据库地址**：localhost或具体IP地址
- **端口**：MySQL默认3306
- **用户名**：root或自定义用户名
- **密码**：设置的数据库密码
- **数据库名称**：项目数据库名称
- **连接字符串**：提供完整的数据库连接字符串示例

### 8. Git提交和打标签
```bash
git add .
git commit -m "[阶段5] 构建部署完成

- 配置生产环境构建
- 生成部署配置
- 准备CI/CD配置
- 输出部署文档和运维指南"
git tag v1.0-release
```
