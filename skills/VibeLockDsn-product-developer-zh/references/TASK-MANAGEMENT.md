# 任务进度管理规范

## 一、进度文件概述

| 文件 | 位置 | 用途 |
|------|------|------|
| dsg-session.md | 项目根目录 | 设计阶段全局进度（由product-manager创建） |
| dev-session.md | 项目根目录 | 开发阶段全局进度、模块列表、当前状态 |
| dev-task.md | 模块子目录 | 模块开发进度、步骤状态、执行摘要 |

## 二、dev-session.md 规范

### 文件位置
- 必须存储在项目根目录（与 dsg-session.md 同级）

### 核心字段
```markdown
## 当前状态（快速定位）
- 当前阶段：全局开发/模块开发/集成交付
- 当前模块：[模块名称] 或 null
- 当前步骤：[步骤名称]

## 模块清单
| 序号 | 模块名称 | 优先级 | 设计状态 | 开发状态 | 设计文件夹 |
|------|---------|--------|---------|---------|-----------|
```

### 完整格式
```markdown
# 产品开发全局进度

## 当前状态
- 当前阶段：全局开发/模块开发/集成交付
- 当前模块：[模块名称] 或 null
- 当前步骤：[步骤名称]
- 创建时间：[创建日期]
- 最后更新：[更新日期]

## 基本信息
- 产品名称：[产品名称]
- 技术栈：[前端]/[后端]/[数据库]
- 设计文件夹：[设计文件夹路径]

## 模块清单

| 序号 | 模块名称 | 优先级 | 设计状态 | 开发状态 | 设计文件夹 | 开始时间 | 完成时间 |
|------|---------|--------|---------|---------|-----------|---------|---------|
| 1 | [模块1] | P0 | 已完成 | 未开始/进行中/已完成 | [路径] | [日期] | [日期] |
| 2 | [模块2] | P1 | 已完成 | 未开始/进行中/已完成 | [路径] | [日期] | [日期] |

## 全局开发状态
| 步骤 | 名称 | 状态 | Git标签 |
|------|------|------|---------|
| 1 | 项目初始化 | 未开始/进行中/已完成 | v0.1-init |
| 2 | 数据库设计 | 未开始/进行中/已完成 | v0.2-database |

## 执行摘要
### [日期] - [操作描述]
- 决策：[关键决策]
- 变更：[重要变更]
- 确认：[用户确认要点]
```

## 三、dev-task.md 规范

### 文件位置
- 路径格式：`[设计文件夹]/[模块名称]/dev-task.md`

### 核心字段
```markdown
## 当前状态（快速定位）
- 当前步骤：[步骤编号].[步骤名称]
- 步骤状态：未开始/进行中/已完成
```

### 完整格式
```markdown
# [模块名称] 开发任务进度

## 当前状态
- 当前步骤：[步骤编号].[步骤名称]
- 步骤状态：未开始/进行中/已完成
- 开始时间：[开始日期]
- 最后更新：[更新日期]

## 基本信息
- 模块名称：[模块名称]
- 优先级：P0/P1/P2
- 所属产品：[产品名称]
- 设计文件夹：[设计文件夹路径]

## 步骤清单

| 步骤 | 名称 | 状态 | 核心交付物 | Git标签 |
|------|------|------|-----------|---------|
| 1 | 模块初始化 | 已完成 | dev-task.md | - |
| 2 | 后端服务开发 | 进行中 | API接口、业务逻辑 | v[X].1-backend |
| 3 | 前端界面开发 | 未开始 | 页面、UI组件 | v[X].2-frontend |
| 4 | 模块测试 | 未开始 | 测试报告 | v[X].3-tested |
| 5 | 模块完成确认 | 未开始 | 更新进度 | - |

## 步骤详情

### 步骤1: 模块初始化
- **状态**: 已完成
- **核心目标**: 建立模块开发环境
- **交付物**: dev-task.md
- **执行摘要**:
  - [日期] 创建模块开发任务文件
  - [日期] 读取设计产物完成

### 步骤2: 后端服务开发
- **状态**: 进行中
- **核心目标**: 实现模块后端服务
- **交付物**: API接口、业务逻辑
- **执行摘要**:
  - [日期] 开始设计API接口
  - [日期] 完成用户管理API

### 步骤3: 前端界面开发
- **状态**: 未开始
- **核心目标**: 实现模块前端界面
- **交付物**: 页面、UI组件
- **执行摘要**: -

### 步骤4: 模块测试
- **状态**: 未开始
- **核心目标**: 验证模块功能
- **交付物**: 测试报告
- **执行摘要**: -

## 交付物清单
- [ ] API接口文档
- [ ] 后端业务逻辑
- [ ] 前端页面
- [ ] UI组件
- [ ] 测试报告
```

## 四、进度更新规则

### 更新时机

| 事件 | 更新文件 | 更新内容 |
|------|---------|---------|
| 开始开发项目 | dev-session.md | 创建文件，初始化基本信息 |
| 全局步骤完成 | dev-session.md | 更新全局开发状态、当前步骤 |
| 开始新模块 | dev-session.md | 更新当前模块、模块状态 |
| 模块步骤完成 | dev-task.md | 更新步骤状态、执行摘要 |
| 模块完成 | dev-session.md | 更新模块状态、当前模块为null |

### 更新操作

#### 步骤开始时
```markdown
## 当前状态
- 当前步骤：[步骤编号].[步骤名称]
- 步骤状态：进行中
```

#### 步骤完成时
```markdown
## 当前状态
- 当前步骤：[下一步骤编号].[下一步骤名称]
- 步骤状态：未开始

### 步骤X: [步骤名称]
- **状态**: 已完成
- **执行摘要**:
  - [日期] [关键操作]
  - [日期] [用户确认内容]
```

## 五、与设计进度的关联

### 读取设计进度
- 开发前必须读取 dsg-session.md 确认设计状态
- 只能开发"设计状态：已完成"的模块
- 按设计完成顺序开发模块

### 进度同步
- dev-session.md 的模块列表来源于 dsg-session.md
- 模块优先级遵循 dsg-session.md 中的定义
- 设计文件夹路径来源于 dsg-session.md

## 六、快速恢复指南

### 恢复操作
1. 读取 dsg-session.md 确认设计进度
2. 读取 dev-session.md 的"当前状态"部分
3. 根据"当前阶段"确定恢复位置
4. 如在模块开发阶段，读取对应模块的 dev-task.md
5. 根据"当前步骤"确定具体恢复点
6. 继续执行当前步骤

### 恢复示例
```
dsg-session.md 显示：
- 用户管理：设计已完成
- 权限管理：设计已完成
- 审批流程：设计进行中

dev-session.md 显示：
- 当前阶段：模块开发
- 当前模块：用户管理
- 当前步骤：后端服务开发

恢复操作：
1. 确认用户管理模块设计已完成
2. 加载 用户管理/dev-task.md
3. 确认步骤2"后端服务开发"的状态
4. 继续执行后端开发任务
```
