# 协调实现流程指南

## 概述

本指南介绍如何使用协调实现流程，确保在生成前后端代码之前强制进行规划，并集成架构师提示词。

## 核心技能

### 1. 规划技能 (sz-implementation-planning)
- **功能**：在代码生成之前强制进行实现规划
- **输入**：前后端架构设计文档、数据库设计文档、API接口文档
- **输出**：前端实现规划文档、后端实现规划文档
- **关键特性**：分析文档合理性、制定详细实现计划、风险评估

### 2. 前端实现技能 (sz-frontend-implementation)
- **功能**：根据规划文档和架构文档生成前端代码
- **集成**：强制使用`Frontend-Architect.md`提示词
- **强制规划**：检查规划文档，确保规划先于实现

### 3. 后端实现技能 (sz-backend-implementation)
- **功能**：根据规划文档和架构文档生成后端代码
- **集成**：强制使用`Backend-Architect.md`提示词
- **强制规划**：检查规划文档，确保规划先于实现

### 4. 协调实现技能 (sz-coordinated-implementation)
- **功能**：协调整个规划-实现流程
- **作用**：确保规划阶段在前，代码生成在后
- **集成**：协调所有相关技能和提示词

## 使用流程

### 标准流程
```
1. 用户请求实现前后端代码
2. 调用sz-coordinated-implementation技能
3. 强制调用sz-implementation-planning进行规划
4. 规划文档得到批准
5. 调用sz-frontend-implementation生成前端代码
6. 调用sz-backend-implementation生成后端代码
7. 进行集成验证
```

### 详细步骤

#### 步骤1：启动协调流程
```
用户：请为微平台项目实现前后端代码
```

#### 步骤2：规划阶段（强制）
协调技能会自动：
1. 检查输入文档：
   - `02技术设计文档/微平台前端技术架构设计文档.md`
   - `02技术设计文档/微平台后端技术架构设计文档.md`
   - `02技术设计文档/微平台数据库设计文档.md`
   - `02技术设计文档/微平台API接口文档.md`

2. 调用Feature-Planner技能（使用`/Users/shenzhiqiang/.claude/skills/Feature-Planner.md`提示词）

3. 输出规划文档：
   - `02技术设计文档/微平台前端实现规划文档.md`
   - `02技术设计文档/微平台后端实现规划文档.md`

#### 步骤3：前端实现阶段
协调技能调用前端实现技能，该技能会：
1. 检查规划文档是否存在
2. 使用`Frontend-Architect.md`提示词指导实现
3. 根据规划文档生成前端代码
4. 输出到`04frontend/`目录

#### 步骤4：后端实现阶段
协调技能调用后端实现技能，该技能会：
1. 检查规划文档是否存在
2. 使用`Backend-Architect.md`提示词指导实现
3. 根据规划文档生成后端代码
4. 输出到`05backend/`目录

#### 步骤5：集成验证
协调技能进行：
1. 接口联调检查
2. 依赖关系验证
3. 整体质量评估

## 架构师提示词集成

### Frontend-Architect.md 集成点
前端实现技能必须遵循以下原则：
1. **用户界面开发**：创建响应式、可访问的用户界面
2. **组件架构设计**：构建可复用、可维护的组件层次结构
3. **状态管理策略**：实现适当的状态管理解决方案
4. **性能优化要求**：优化包大小、渲染性能和资源交付
5. **代码质量标准**：确保代码质量、测试覆盖率和安全性

### Backend-Architect.md 集成点
后端实现技能必须遵循以下原则：
1. **API设计标准**：遵循RESTful API设计最佳实践
2. **服务架构原则**：构建可扩展、可维护的服务层
3. **数据库架构要求**：设计规范化的数据库模式，优化查询性能
4. **系统安全标准**：实现身份验证、授权和数据保护
5. **运维监控要求**：确保系统可观测性和故障恢复能力

## 强制规划机制

### 检查机制
前后端实现技能在工作流程开始时检查：
1. 规划文档是否存在
2. 规划文档是否得到批准
3. 规划方案是否合理

### 处理流程
```
if (规划文档不存在) {
    调用Feature-Planner技能
    生成规划文档
    等待用户批准
}
if (规划文档已批准) {
    根据规划文档进行实现
} else {
    暂停实现，等待批准
}
```

## 错误处理

### 常见错误及处理
1. **规划文档不存在**
   - 处理：自动调用规划技能生成文档

2. **规划文档未批准**
   - 处理：暂停实现，等待用户批准

3. **架构文档不完整**
   - 处理：提示用户补充缺失文档

4. **前后端接口不一致**
   - 处理：协调技能进行接口对齐

## 最佳实践

### 规划阶段
1. **详细分析**：充分分析架构文档的合理性和完整性
2. **风险评估**：识别所有潜在的技术风险
3. **明确依赖**：明确前后端实现之间的依赖关系
4. **获得批准**：确保规划方案得到正式批准

### 实现阶段
1. **遵循规划**：严格遵循规划文档进行实现
2. **集成提示词**：确保架构师提示词得到正确应用
3. **质量保证**：在每个阶段进行质量检查
4. **文档同步**：保持代码和文档的一致性

### 协调阶段
1. **流程监控**：监控整个规划-实现流程
2. **问题协调**：及时协调解决技术问题
3. **质量验证**：验证最终实现质量
4. **知识管理**：确保所有文档正确归档

## 扩展和定制

### 自定义规划模板
可以修改`sz-implementation-planning/SKILL.md`中的规划文档模板。

### 添加新的架构原则
可以更新`Frontend-Architect.md`和`Backend-Architect.md`文件，添加新的架构原则。

### 调整强制检查点
可以修改前后端实现技能中的强制规划检查逻辑。

## 总结

协调实现流程通过强制规划阶段和架构师提示词集成，确保了：
1. **规划先行**：代码生成之前必须有详细的实现规划
2. **架构指导**：实现遵循专业的架构原则
3. **质量保证**：通过多阶段检查确保实现质量
4. **知识管理**：完整的文档记录便于后续维护

这个流程特别适合复杂项目的实现，能够有效降低技术风险，提高实现质量。
