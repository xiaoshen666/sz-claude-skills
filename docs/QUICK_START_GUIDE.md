# Obsidian与自定义Skills集成快速启动指南

## 🚀 快速开始

### 1. 启动一个项目
```
用户：请帮我分析微平台项目的需求
```
Claude会自动调用`sz-requirements-analysis-design` skill，并询问：
- 项目名称：微平台
- 是否需要Obsidian格式输出：是（默认）

### 2. 查看生成的文档
skill会自动创建：
- **原始文档**：`01需求文档/微平台需求概要文档.md`
- **Obsidian文档**：`微平台-知识库/01-需求文档/微平台需求概要文档.md`

### 3. 继续后续环节
```
用户：请设计技术架构
用户：请创建UI原型
用户：请实现前端代码
...
```
每个环节都会自动调用相应的skill，并生成Obsidian格式文档。

## 📁 文件夹结构

```
项目根目录/
├── 01需求文档/                    # 原始需求文档
├── 02技术设计文档/                # 原始技术设计文档
├── 03原型/                       # 原始原型文档
├── 04frontend/                   # 前端代码
├── 05backend/                    # 后端代码
├── image/                        # 图片和附件
└── {项目名称}-知识库/             # Obsidian知识库
    ├── 00-项目概览/
    ├── 01-需求文档/
    ├── 02-技术设计/
    ├── 03-设计文档/
    ├── 04-开发文档/
    ├── 05-部署运维/
    ├── 06-评审记录/
    ├── 07-会议记录/
    ├── 08-参考资料/
    ├── attachments/
    └── {项目名称}-文档管理.base   # Bases视图
```

## 🔧 核心Skills列表

| Skill名称 | 功能描述 | 输出文档 |
|----------|---------|---------|
| **sz-requirements-analysis-design** | 需求分析设计 | 需求概要文档、需求详细设计文档 |
| **sz-architecture-design** | 技术架构设计 | 前端/后端/整体架构文档、技术选型矩阵 |
| **sz-database-api-design** | 数据库和接口设计 | 数据库设计、ER图、SQL脚本、API接口文档 |
| **sz-ui-prototype-design** | UI原型设计 | 站点地图、页面流程、交互规范、原型设计 |
| **sz-frontend-implementation** | 前端代码实现 | 前端项目配置、组件文档、页面文档 |
| **sz-backend-implementation** | 后端代码实现 | 后端项目配置、业务逻辑文档、API实现文档 |
| **sz-integration-deployment** | 集成部署 | 项目集成文档、部署实施文档、运维文档 |
| **sz-deliverables-review** | 交付物检查 | 评审报告、问题清单、质量指标 |
| **sz-obsidian-integration** | Obsidian集成 | 知识库创建、文档转换、Bases视图 |

## ⚙️ 配置选项

### 1. 关闭Obsidian格式输出
在skill询问时选择"否"，或修改skill文件中的默认设置。

### 2. 自定义知识库结构
修改`sz-obsidian-integration/obsidian-template.md`中的目录映射。

### 3. 调整文档模板
编辑`sz-obsidian-integration/obsidian-template.md`中的frontmatter模板。

## 🛠️ 故障排除

### 问题1：skill未正确调用
**解决**：确保skill名称正确，格式为`sz-{功能名称}`

### 问题2：文档位置错误
**解决**：检查文件夹结构是否已创建

### 问题3：Obsidian格式错误
**解决**：查看`sz-obsidian-integration` skill的模板文件

### 问题4：链接无效
**解决**：确保所有相关文档已生成
