在 Mac 上使用 Claude Code 时，`.claude` 文件夹就是你的“指挥中心”。它不仅决定了 AI 如何理解你的项目，还能通过自定义配置极大地提升你的工作效率。

这个文件夹其实分为两个层级：**项目级**（位于项目根目录，团队共享）和 **全局级**（位于 `~/.claude/`，仅你个人使用）。

以下是各个配置目录的详细介绍，以及如何通过自定义 Skills 来打造你的高效工作流。

### 📂 .claude 配置目录详解

这个文件夹里的每个文件和目录都有特定的职责，理解它们能帮你更好地掌控 AI：

| 目录/文件 | 作用描述 | 适用场景 |
| :--- | :--- | :--- |
| **CLAUDE.md** | **核心大脑**。这是最重要的文件，每次会话启动时都会被读取。用于定义项目背景、技术栈、常用命令和核心规则。 | 团队共享的项目规范、架构说明。 |
| **skills/** | **可复用工作流**。存放复杂的任务脚本。与命令不同，Skills 是“按需加载”的，只有当任务匹配描述时才会激活，不占用日常上下文。 | 代码审查、部署流程、复杂的代码重构。 |
| **commands/** | **快捷指令**。存放自定义的斜杠命令（如 `/fix`）。当你输入指令时才会触发。 | 快速生成提交信息、每日站会辅助。 |
| **rules/** | **模块化规则**。用于拆分庞大的 `CLAUDE.md`。可以根据文件路径或主题（如测试、API）自动加载特定规则。 | 针对特定目录（如 `src/api`）的编码规范。 |
| **agents/** | **专家角色**。定义专门的子代理，拥有独立的提示词和工具集，用于处理特定领域的复杂任务。 | 专门的安全审计员、代码审查员。 |
| **settings.json**| **权限控制**。定义 Claude 能做什么（如允许运行哪些命令、读取哪些文件）。 | 限制 AI 修改生产环境配置或敏感文件。 |

### 🚀 如何自定义 Skills 提高工作效率

Skills 是提升效率的利器。你可以把它理解为给 Claude 的一本“操作手册”，教它如何处理重复性高或流程复杂的任务。

#### 1. 创建你的第一个 Skill

你可以在全局目录 `~/.claude/skills/` 中创建个人通用的技能，也可以在项目目录 `.claude/skills/` 中创建项目专用的技能。

**步骤如下：**

1.  **新建文件夹**：在 `~/.claude/skills/` 下创建一个新文件夹，命名必须使用 **kebab-case**（如 `code-review-skill`）。
2.  **创建核心文件**：在文件夹内创建一个名为 `SKILL.md` 的文件（注意全大写）。

#### 2. 编写 SKILL.md 内容

这个文件包含两部分：顶部的 **YAML 元数据**（告诉 Claude 何时使用它）和 **正文**（告诉 Claude 怎么做）。

**示例：一个自动代码审查的 Skill**

```markdown
---
name: code-reviewer
description: 当用户要求审查代码、检查代码质量或寻找潜在 Bug 时使用此技能。
---

### 角色定位
你是一位拥有 10 年经验的资深代码审查专家，专注于代码的可读性、性能和安全性。

### 核心规则
1.  **安全性优先**：检查是否有 SQL 注入风险、敏感信息泄露（如 API Key）。
2.  **性能检查**：识别低效的循环或内存泄漏风险。
3.  **风格一致性**：确保代码符合项目的 ESLint 和 Prettier 配置。

### 工作流程
1.  阅读用户提供的代码片段或文件路径。
2.  分析代码逻辑。
3.  按照以下格式输出报告：
    -   **🔴 严重问题**：必须修复的 Bug 或安全漏洞。
    -   **🟡 建议改进**：代码优化建议。
    -   **🟢 亮点**：写得好的地方。

### 注意事项
-   不要修改代码，只提供审查意见。
-   如果代码没有明显问题，请回复“代码质量良好，符合规范”。
```

#### 3. 进阶技巧

*   **渐进式披露**：Claude 平时只会读取 `description`，只有当你触发相关任务时，才会加载完整的 `SKILL.md` 内容。这意味着你可以写非常详细的指令，而不用担心浪费上下文窗口。
*   **配合 MCP 工具**：Skills 解决“怎么做”，MCP 解决“用什么”。你可以在 Skill 中指示 Claude 使用 MCP 工具（如数据库查询工具）来辅助完成任务。
*   **利用技能-创造者（Skill Creator）**：如果你使用 Claude 网页版，可以在设置中找到 `skill-creator` 工具，它会通过对话引导你生成 `SKILL.md` 文件。

### 💡 提效建议

1.  **区分个人与项目**：将通用的技能（如“写单元测试”、“解释代码”）放在 `~/.claude/skills/`，这样所有项目都能用；将特定业务逻辑（如“生成特定格式的 API 文档”）放在项目级的 `.claude/skills/`。
2.  **模块化规则**：不要把所有规则都塞进 `CLAUDE.md`。如果文件超过 200 行，Claude 的遵循能力会下降。请使用 `rules/` 文件夹将规则拆分，例如 `rules/testing.md` 专门管测试规范。
3.  **设置权限**：在 `settings.json` 中明确允许或拒绝某些操作（例如拒绝读取 `.env` 文件），这能让你更放心地让 AI 自动执行任务。
